# PR Review Checklist

## Correctness
- [ ] Change solves the stated problem
- [ ] Edge cases are handled
- [ ] Error paths are explicit and safe

## Security and Privacy
- [ ] Inputs are validated/sanitized
- [ ] Auth/authz behavior is preserved
- [ ] No secrets or sensitive data exposure

## Maintainability
- [ ] Naming and structure are clear
- [ ] Complexity is justified and bounded
- [ ] Documentation/comments updated where needed

## Tests
- [ ] Unit tests added/updated
- [ ] Integration/e2e coverage considered
- [ ] Failing-path tests included where relevant

## Operational Readiness
- [ ] Logging/metrics are sufficient
- [ ] Backward compatibility considered
- [ ] Rollback strategy exists for risky changes

## Final Recommendation
- [ ] Approve
- [ ] Approve with follow-ups
- [ ] Request changes
